Part 1: Amazon Seller Central

Setting up Amazon SP-API access

‚è±Ô∏è 45 min + 2-3 days

1 Access Amazon Developer Central

  1. Open your web browser and go to: https://developer.amazonservices.com
  2. Click the "Sign in" button in the top right corner
  3. Sign in using the same email and password you use for Amazon Seller Central
  4. Verify your identity if prompted (may require 2-step verification)

2 Register as a Developer

  1. After signing in, you'll see a registration page
  2. Fill in the following information:
    • Developer Name: Your business name or your name
    • Email Address: Your business email (receives important notifications)
    • Phone Number: Your business phone number
    • Company Name: Your registered business name
    • Website: Your business website or Amazon storefront URL
  3. Read and accept the Amazon Services Terms of Use
  4. Click "Submit" or "Register"
‚úÖ You're now registered as an Amazon developer!

3 Create a Developer Application

  1. Go to the "Apps & Services" menu at the top
  2. Click on "Manage Your Apps"
  3. Click the orange "Add New App" button
  4. Fill in the application details:
Application Name: Warehouse Management Integration Application Description: Integration for warehouse management system to sync orders and inventory OAuth Login URI: https://localhost OAuth Redirect URI: https://localhost/callback
‚ÑπÔ∏è Note: The URLs are temporary and will be updated later by your developer
  1. Click "Save" or "Create App"

4 Get Your API Credentials

  1. After creating the app, click on "Edit App" or your app name
  2. Look for the "LWA Credentials" section (LWA = Login with Amazon)
  3. You'll see two important credentials:
    • Client ID: A long string of letters and numbers
    • Client Secret: Click "View" to see it
‚ö†Ô∏è IMPORTANT - Save These Immediately! Copy and paste these credentials somewhere safe (use a password manager or secure note app):
Amazon Client ID: [paste your Client ID here] Amazon Client Secret: [paste your Client Secret here]

5 Enable Required API Scopes

  1. On the same page, look for "Roles" or "API Permissions"
  2. Make sure these are checked/enabled:
    • ‚úÖ Orders API (to fetch orders)
    • ‚úÖ Feeds API (to update inventory)
    • ‚úÖ Notifications API (for real-time updates)
    • ‚úÖ Merchant Fulfillment API (to mark orders as shipped)
  3. Click "Save" or "Update"

6 Get Your Refresh Token

This is the trickiest part. Choose one of these options:

üìû Option A: Request from Amazon Support (Easier)
  1. Go to Amazon Seller Central
  2. Click "Help" at the top
  3. Search for "SP-API refresh token"
  4. Contact support and say: "I need a refresh token for my SP-API application to integrate with my warehouse system"
  5. They will guide you through the authorization process
  6. Save the refresh token they provide
üíª Option B: Use OAuth Flow (For Developers)
  1. Share your Client ID with your developer
  2. They will create an authorization URL for you
  3. Visit that URL, log in to Amazon, and authorize the app
  4. You'll receive a refresh token
  5. Save it securely
‚ö†Ô∏è Save All This Information:
Amazon Refresh Token: [paste here] Amazon Seller ID: [found in Seller Central > Settings > Account Info] Amazon Marketplace ID: [usually ATVPDKIKX0DER for US]

7 Enable Notifications (Optional but Recommended)

  1. In your Amazon Developer Console, go to "Notifications"
  2. Click "Create Destination"
  3. Select "SQS" (Simple Queue Service)
  4. Name it: warehouse-notifications
  5. Save the SQS Queue URL provided
  6. Share this URL with your developer - they'll need it to receive real-time order updates

8 Test Your Setup

  1. In the Developer Console, look for "Test" or "Sandbox"
  2. Try making a test API call (your developer can help with this)
  3. If it works, you're all set! ‚úÖ
‚è∞ Approval Wait Time: Amazon may take 2-3 business days to fully approve your developer account and enable API access. You'll receive an email when it's ready.

Part 2: eBay Developer Program

Setting up eBay API access

‚è±Ô∏è 30 min + 1-2 days

1 Access eBay Developer Program

  1. Go to: https://developer.ebay.com
  2. Click "Register" or "Sign In" in the top right
  3. Use your regular eBay seller account credentials to log in
  4. If you don't have a developer account yet, you'll be prompted to create one

2 Register as a Developer

  1. Click "My Account" after logging in
  2. Fill in the registration form:
    • First Name & Last Name: Your name
    • Email: Your business email
    • Company Name: Your business name (can be individual)
    • Country: Your country
    • Phone: Your phone number
  3. Accept the Developer Terms of Use
  4. Click "Continue" or "Register"

3 Create Production Keys

  1. Go to "My Account" ‚Üí "Application Keysets"
  2. You'll see options for Sandbox and Production
  3. Click "Create Production Keyset"
  4. Fill in the application details:
Application Title: Warehouse Management System Application Description: Integration for syncing orders and inventory between eBay and warehouse Application Type: Web Application Grant Type: Authorization Code (for OAuth)
  1. Click "Create Keyset"

4 Copy Your Credentials

After creating the keyset, you'll see three important credentials:

‚ö†Ô∏è SAVE THESE IMMEDIATELY! (They may not be shown again)
eBay App ID (Client ID): [paste here] eBay Dev ID: [paste here] eBay Cert ID (Client Secret): [paste here]
‚ÑπÔ∏è How to find them later:
  1. Go to "My Account" ‚Üí "Application Keysets"
  2. Click on your application name
  3. Your credentials will be displayed (you may need to click "View" on the Cert ID)

5 Configure OAuth Scopes

  1. In your application settings, look for "OAuth Scopes" or "Permissions"
  2. Make sure these are enabled:
    • ‚úÖ https://api.ebay.com/oauth/api_scope/sell.inventory (manage inventory)
    • ‚úÖ https://api.ebay.com/oauth/api_scope/sell.fulfillment (manage orders)
    • ‚úÖ https://api.ebay.com/oauth/api_scope/sell.account (account access)
  3. Click "Save Changes"

6 Configure Redirect URLs

  1. Still in your application settings, find "OAuth Redirect URIs"
  2. Click "Add Redirect URI"
  3. For now, enter:
https://localhost/auth/ebay/callback
  1. Click "Save"
  2. Later, your developer will provide you with the actual URL to update this

7 Enable Required Scopes

  1. In your application settings, look for "OAuth Scopes" or "Permissions"
  2. Make sure these are enabled:
    • ‚úÖ https://api.ebay.com/oauth/api_scope/sell.inventory (manage inventory)
    • ‚úÖ https://api.ebay.com/oauth/api_scope/sell.fulfillment (manage orders)
    • ‚úÖ https://api.ebay.com/oauth/api_scope/sell.account (account access)
  3. Click "Save Changes"

8 Get Your OAuth User Token

This requires authorization:

Method 1: Developer Assisted
  1. Your developer will create an authorization URL using your App ID
  2. Visit that URL in your browser
  3. You'll see an eBay login/consent screen
  4. Click "Agree" or "Authorize"
  5. You'll be redirected to a page with a code
  6. Share that code with your developer, who will exchange it for an access token
Method 2: Alternative Method
  1. In eBay Developer Program, go to "User Tokens"
  2. Click "Generate User Token"
  3. Select your application
  4. Choose the same scopes as above
  5. Click "Generate Token"
  6. Copy and save the token
‚ö†Ô∏è Save This:
eBay OAuth User Token: [paste here] eBay OAuth Refresh Token: [paste here - if provided] Token Expiration Date: [paste here]

9 Set Up Notifications (Optional)

  1. Go to "Notifications" in the Developer Program
  2. Click "Create Notification"
  3. Select "Platform Notifications"
  4. Choose notification types:
    • ‚úÖ Order created
    • ‚úÖ Order updated
    • ‚úÖ Item sold
  5. Enter your webhook URL (your developer will provide this)
  6. Click "Subscribe"

10 Get Your Seller Account Information

You'll also need these details:

  1. Go to your regular eBay Seller Account
  2. Click "Account" ‚Üí "Personal Information"
  3. Find and save:
eBay User ID: [your seller username] eBay Site ID: [e.g., 0 for US, 3 for UK, 77 for Germany]
‚è∞ Approval Wait Time: eBay typically approves developer accounts within 1-2 business days. You'll receive an email confirmation.

Part 3: Shopify Setup

Setting up Shopify Admin API

‚è±Ô∏è 30 min - Instant!

1 Access Your Shopify Admin

  1. Log in to your Shopify store at: https://[your-store-name].myshopify.com/admin
  2. Replace [your-store-name] with your actual store name
  3. Enter your Shopify admin credentials

2 Enable Custom App Development

  1. In the left sidebar, click "Settings" at the bottom
  2. Click on "Apps and sales channels"
  3. Click "Develop apps" at the top
  4. If you see a message saying "App development is disabled":
    • Click "Allow custom app development"
    • Confirm by clicking "Allow custom app development" again

3 Create a Custom App

  1. Click "Create an app" button
  2. Enter the app name:
App Name: Warehouse Management Integration
  1. In the "App developer" field, select yourself or your primary admin email
  2. Click "Create app"

4 Configure API Scopes (Permissions)

  1. After creating the app, click the "Configuration" tab
  2. Under "Admin API integration", click "Configure"
  3. Enable these permissions:
    • ‚úÖ Orders: read_orders, write_orders
    • ‚úÖ Inventory: read_inventory, write_inventory
    • ‚úÖ Products: read_products, write_products (optional)
    • ‚úÖ Fulfillment: read_fulfillments, write_fulfillments
  4. Click "Save" at the bottom

5 Install the App

  1. Click on the "API credentials" tab
  2. Click "Install app" button
  3. You'll see a confirmation popup
  4. Click "Install" to confirm

6 Get Your API Credentials

After installing, you'll see your credentials:

‚ö†Ô∏è COPY THESE IMMEDIATELY! The Admin API access token is only shown once!
Shopify Store URL: [your-store-name].myshopify.com Shopify API Key: [paste here] Shopify API Secret Key: [click "Reveal" to see it, then paste here] Admin API Access Token: [paste here - VERY IMPORTANT!] API Version: 2024-01 (or latest version shown)
‚ö†Ô∏è If you miss copying the access token: You'll need to uninstall and reinstall the app to see it again.

7 Enable Webhooks

  1. In your app, go to "Configuration" tab
  2. Scroll to "Webhooks" section
  3. Click "Create webhook"
  4. Create webhooks for these events:
Order Webhooks:
  • Event: orders/create
  • URL: [Your developer will provide this]
  • Click "Save"
  • Event: orders/updated
  • URL: [Same as above]
  • Click "Save"
Inventory Webhooks:
  • Event: inventory_levels/update
  • URL: [Same as above]
  • Click "Save"
  1. For now, you can use a temporary URL like https://example.com/webhook
  2. Your developer will give you the real webhook URL to update later

8 Test Your Setup

  1. Go to your Shopify Admin ‚Üí "Orders"
  2. Create a test order (you can mark it as test order)
  3. Your developer can verify if the order data can be accessed via the API
  4. ‚úÖ If successful, you're all set!

9 Note Your Shop Information

You'll also need these details:

Shop ID: [found in Settings ‚Üí General ‚Üí bottom of page] Shop Domain: [your-store-name].myshopify.com Primary Location ID: [Settings ‚Üí Locations ‚Üí click your location ‚Üí ID in URL]
‚è∞ Approval Wait Time: None! Shopify custom apps work immediately. ‚úÖ

üîê Security Best Practices

‚úÖ DO:
  • Store credentials in a password manager (LastPass, 1Password, etc.)
  • Only share credentials with trusted developers via secure channels
  • Use password-protected documents or encrypted email
  • Regularly review authorized apps in each marketplace
  • Enable two-factor authentication on all marketplace accounts
‚ùå DON'T:
  • Share credentials via plain email or text message
  • Save credentials in unsecured documents
  • Share your master login passwords (API tokens are separate)
  • Authorize apps you don't recognize
  • Take screenshots of credentials and store them unencrypted

üÜò Common Issues & Getting Help

Issue: "I can't find the Developer section on Amazon"
Solution: Make sure you're on https://developer.amazonservices.com (NOT regular Amazon.com or Seller Central). It's a separate portal.
Issue: "I accidentally closed the Shopify token page"
Solution: Go back to Apps ‚Üí Your app ‚Üí API credentials ‚Üí Click "Uninstall app" ‚Üí Click "Install app" again ‚Üí You'll see the token again!
Issue: "My tokens keep expiring"
Solution: Amazon refresh tokens don't expire. eBay tokens expire every 18 months (you'll need to regenerate). Shopify tokens don't expire unless you uninstall the app.
